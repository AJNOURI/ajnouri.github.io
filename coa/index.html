Identity

Having an old rc file and without downloading rc file from project > Access and security > API access >, make sure to build rc file for both tenants admin & demo

Go to identity > project, get project id from old admin-openrc.sh, replace tentant id with project admin id to build admin credential file and demo credential file and finally source it: source demorc.sh

Creating a project named "new-project1", a user "new-user1" with password "new-user1", and assign a new role role "manager" to that user.

openstack project create "project1"
openstack project list
openstack user create "new-user1"
openstack user list
openstack role create "manager"
openstack role list

Compute

From demo account, generate a public keypair named mypubkey1 for use with openstack instances

nova keypair-add key1 key1.pem
chmod 600 key1.pem
nova keypair-list

From admin account, create a new flavor named m1.extra_tiny with:

  • RAM: 64mb
  • Root disk size: 0
  • cpu: 1

. admin_rc.sh
nova flavor-lis
nova flavor-create m1.extra_tiny auto 64 0 1 --rxtx-factor 1.0

Allow the tenant (project) "demo" to access the flavor

openstack project list
nova flavor-access-add m1.extra_tiny


Additional commands:
nova flavor-list
nova flavor-delete {flavorid}

Add new rules to the default security group "default" to allow access instances from internet through SSH, http and ICMP.

nova secgroup-list
nova secgroup-list-rules default
nova secgroup-add-rule default tcp 22 22 0.0.0.0/0
nova secgroup-add-rule default tcp 80 80 0.0.0.0/0
nova secgroup-add-rule default icmp -1 -1 0.0.0.0/0

Provision the following instance:
  • image: cirros-0.3.4-x86_64-uec
  • flavor: m1.tiny
  • keypair: key1
  • security group: default

nova boot instance2 --image cirros-0.3.4-x86_64-uec --flavor m1.tiny --key-name key1 --security-group default

Log to the machine console using the keypair key1

Get the instance IP address from nova list command:
ssh -i key1.pem ubuntu@{ip}

Create a snapshot named "instance2-snapshot" from the running image instance2

nova image-create {instance2} {instance2-snapshot}
nova delete {instance2}
